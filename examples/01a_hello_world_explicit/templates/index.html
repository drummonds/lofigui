01a — Hello World Explicit

Explicit Handlers + Service Worker WASM

Same model as 01, unbundled into HandleRoot / HandleDisplay / HandleCancel

What this demonstrates

  • Explicit route registration — setupRoutes() returns a *http.ServeMux with separate start, display, and cancel handlers
  • Custom template — embedded Go string constant with start/cancel forms, replacing the default template
  • Service worker WASM — same setupRoutes() serves both the HTTP server and go-wasm-http-server in the browser
  • No custom JavaScript — unlike example 01’s app.js (which polls goRender() via setInterval), the service worker intercepts normal HTTP requests

Server-side flow

GET  /       → app.HandleDisplay() — renders template with current buffer
POST /start  → app.HandleRoot()    — resets buffer, starts model, redirects to /
POST /cancel → app.HandleCancel()  — cancels action, redirects to /

The WASM demo below uses a service worker to intercept these same HTTP requests in the browser. Run task go-example:01a to see the server version.

Live Demo (Service Worker WASM)

The service worker loads the Go WASM binary and intercepts all fetch requests. Your browser makes real HTTP requests to /, /start, and /cancel — the service worker routes them to the same Go handlers that run on the server.
