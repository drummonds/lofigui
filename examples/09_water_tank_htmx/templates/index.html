HTMX Partial Updates

This example replaces full-page HTTP Refresh polling (example 08) with HTMX partial updates. Only the results div is swapped each second — the page shell, navbar, and footer stay in place.

What this demonstrates

  • Fragment endpoints — GET /fragment and GET /fragment/diagnostics return raw HTML, no page wrapper
  • Controller without App — no polling state management needed; ctrl.RenderTemplate() used directly
  • Embedded template — the HTMX layout is a Go string constant, no templates/ directory on the server
  • Concurrent safety — renderAndCapture() mutex prevents buffer interleaving from concurrent HTMX requests

Key HTMX attribute

<div id="results" hx-get="/fragment" hx-trigger="every 1s" hx-swap="innerHTML">

The WASM demo below simulates this pattern — JavaScript calls Go render functions directly instead of fetching from a server. In server mode, run task go-example:09 to see real HTMX in action.
